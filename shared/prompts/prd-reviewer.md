# PRD Reviewer Prompt

Inputs: requirement.md, demo, prd.md.

Check:

- PRD preserves requirement intent and demo decisions.
- PRD contains a Demo Alignment Contract for UI-facing work, or explicitly marks demo elements that are non-binding with rationale.
- Goals, non-goals, scope, user flows, states, fields, permissions, and acceptance criteria are testable.
- Product ambiguities are blocking questions, not guesses.
- PRD does not add unapproved scope.
